# Public Contracts

## Overview

The Kids Daily Planner Dashboard is a static web application (no backend API). This folder documents the public interfaces that external consumers (parents, educators, analytics tools) might interact with.

---

## 1. localStorage Data Contract

**Consumer**: Any web application or browser extension that reads/writes kid data

**Note**: Direct localStorage access by external code is **not recommended** for security/privacy reasons. However, the data structures are documented here for transparency and potential future integrations.

### Keys and Format

```typescript
// Daily Schedule (read-only recommended)
Key: `dailySchedule:${YYYY-MM-DD}`
Type: JSON string
Struct: {
  date: string,
  timeBlocks: [{ blockId: number, time: string, activity: string }, ...],
  createdAt: number,
  updatedAt: number
}

// Daily Stars (read-only recommended)
Key: `dailyStars:${YYYY-MM-DD}`
Type: JSON string
Struct: {
  date: string,
  starCount: 0-5,
  recordedAt: number
}

// Global Want List (read-only recommended)
Key: `wantList`
Type: JSON string
Struct: {
  prizes: [{ id: string, name: string, starCost: number, addedAt: number }, ...],
  totalStars?: number
}

// App Metadata
Key: `appMetadata`
Type: JSON string
Struct: {
  version: string,
  lastMigration: number
}
```

### Access Patterns

**Read All Daily Data for a Date**:
```javascript
const dateStr = '2026-05-16';
const schedule = JSON.parse(localStorage.getItem(`dailySchedule:${dateStr}`) ?? '{}');
const stars = JSON.parse(localStorage.getItem(`dailyStars:${dateStr}`) ?? '{}');
const wantList = JSON.parse(localStorage.getItem('wantList') ?? '{"prizes":[]}');
```

**Write Daily Stars** (not recommended; app manages this):
```javascript
const dateStr = '2026-05-16';
const starsData = { date: dateStr, starCount: 4, recordedAt: Date.now() };
localStorage.setItem(`dailyStars:${dateStr}`, JSON.stringify(starsData));
```

## 2. Static HTML Export Contract

**Consumer**: CDN hosting, static site deployers (Netlify, Vercel, GitHub Pages)

**Entry Point**: `out/index.html` (generated by `next export`)

### Expected Structure

```text
out/
├── index.html             # Main entry point (all pages pre-rendered to static HTML)
├── favicon.ico            # Application icon
├── _next/
│   ├── static/
│   │   ├── css/           # CSS bundles (*.css)
│   │   ├── js/            # JavaScript chunks (*.js)
│   │   └── [hash]/        # Immutable asset cache
│   └── data/              # Build metadata
└── public/                # Static assets (copied from source `/public/`)
    ├── emoji/
    └── fonts/
```

### Deployment Requirements

1. **Web Server Config**:
   - Serve `index.html` for all route requests (single-page app)
   - Set cache headers: `Cache-Control: public, max-age=31536000, immutable` for `_next/static/`
   - Set cache headers: `Cache-Control: public, max-age=3600` for root `/index.html` (refresh daily)

2. **CORS Headers** (if serving to cross-origin requests):
   ```
   Access-Control-Allow-Origin: *
   Access-Control-Allow-Methods: GET, HEAD, OPTIONS
   ```

3. **Content-Type Headers**:
   - `.js` → `application/javascript`
   - `.css` → `text/css`
   - `.woff2` → `font/woff2`

### Example Deployment (Netlify `netlify.toml`)

```toml
[build]
  command = "npm run build && npm run export"
  publish = "out"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

### Example Deployment (Vercel `vercel.json`)

```json
{
  "buildCommand": "npm run build && npm run export",
  "outputDirectory": "out",
  "env": {
    "NODE_ENV": "production"
  }
}
```

## 3. Browser Support Contract

**Consumer**: QA, testers, DevOps (deployment verification)

### Minimum Browser Versions

| Browser | Version | Notes |
|---------|---------|-------|
| Chrome/Chromium | 90+ | ES2021 support; localStorage required |
| Firefox | 88+ | ES2021 support; localStorage required |
| Safari | 14+ | ES2021 support; localStorage required |
| Edge | 90+ | ES2021 support; localStorage required |
| Mobile Safari (iOS) | 14+ | Responsive design optimized |
| Chrome Mobile | 90+ | Responsive design optimized |

### Feature Requirements

- **localStorage API**: Required for data persistence
- **ES2021 JavaScript**: Required for modern syntax (optional chaining, nullish coalescing, etc.)
- **CSS Grid & Flexbox**: Required for responsive layout
- **CSS Custom Properties**: Required for theming (--color-*, --size-*, etc.)
- **CSS Media Queries**: Required for mobile-first responsive design

### Progressive Degradation

- **Without JavaScript**: Schedule renders as static HTML; date picker uses `<input type="date">`; star input uses `<input type="number"`; no animations
- **Without localStorage**: Warn user; data lost on page reload (app remains functional but non-persistent)
- **Without CSS Grid**: Fallback to flexbox layouts; single-column on older browsers

## 6. Data Privacy & Security Contract

**Consumer**: Parents, guardians, compliance officers

### Data Storage Location

- **All user data**: Stored locally in browser `localStorage` **only**
- **No server transmission**: Data never leaves the child's device
- **No analytics tracking**: No third-party tracking cookies or scripts
- **No authentication required**: Open access (can be restricted via parental controls or URL password)

### Data Retention

- **Schedule Data**: Persists indefinitely until user deletes via browser settings
- **Stars Data**: Persists indefinitely until user deletes via browser settings
- **Want List**: Persists indefinitely until user deletes via browser settings
- **Clear Data**: User can clear all data via "Settings" → "Clear All Data" (or browser localStorage clear)

### Security Considerations

- **Input Validation**: All user inputs validated client-side; no XSS vectors
- **localStorage Isolation**: Data isolated per origin (domain); not accessible cross-domain
- **HTTPS Recommended**: Deployment should use HTTPS to prevent man-in-the-middle attacks (localStorage not encrypted)
- **No PII**: Application doesn't collect name, email, contact info, or location

### COPPA Compliance (US Children's Online Privacy Protection Act)

- **Target Age**: 8–12 years old
- **Parental Consent**: Not required (no data collection or sharing)
- **Data Practices**: No third-party sharing; no behavioral tracking
- **Opt-Out**: Not applicable (no data transmission)

## 7. Extensibility Contract

**Consumer**: Future developers, plugin authors

### Hooks for Future Integration

The app is designed to be extended without modifying core code:

1. **Custom Storage Backend** (future):
   - Replace `localStorage` with IndexedDB or server-based storage
   - Update `lib/storage/*` modules; component interface unchanged

2. **Analytics Integration** (future):
   - Add event logging to custom hook callbacks
   - Non-breaking: no change to UI or data model

3. **Parental Dashboard** (future):
   - Expose public React components for embedding
   - Export custom hooks for external consumption

4. **Mobile App** (future):
   - React Native port shares business logic (custom hooks)
   - Platform-specific UI components only

### Stability Guarantees

- **Major Version**: Breaking changes (e.g., schema migration, hook signature change)
- **Minor Version**: New features, non-breaking additions
- **Patch Version**: Bug fixes, performance improvements

Current version: **1.0.0** (initial release)
